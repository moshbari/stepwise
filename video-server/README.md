# StepWise Video Server

The video rendering backend for StepWise. This is a standalone Node.js service
that runs on the Contabo VPS and turns step-by-step project data into narrated
MP4 videos.

## What's in here

| File                | Purpose                                                                  |
| ------------------- | ------------------------------------------------------------------------ |
| `server.js`         | Main Express server. Receives screenshots + text from the extension, generates TTS audio via OpenAI, renders each step as a video segment with FFmpeg, concatenates into a final MP4. |
| `ghl-uploader.js`   | GoHighLevel upload module. Uses `curl` via `execSync` with multipart form-data (the only method that works with GHL Media API). Auto-places videos in a named folder via `parentId`. |
| `patch-server.js`   | One-time setup script that patches `server.js` to wire up GHL auto-upload after video render. Not used at runtime — historical artifact of the original setup. |
| `package.json`      | Node dependencies.                                                       |

## Where it runs

- **VPS path:** `/opt/stepwise-video/`
- **Port:** `3500`
- **Process manager:** PM2, named `stepwise-video`
- **OS:** Ubuntu 24.04, FFmpeg installed system-wide

## What's NOT in this repo (lives only on the VPS)

Two files are intentionally left out of version control because they contain
secrets or are regeneratable:

- **`ghl-config.json`** — holds the GHL Private Integration Token, location ID, and target folder name. Located at `/opt/stepwise-video/ghl-config.json` with `chmod 600`.
- **`.secret`** — the video server's own API key that the extension must send with every job request. Located at `/opt/stepwise-video/.secret`.
- **`node_modules/`** — run `npm install` inside `/opt/stepwise-video/` to regenerate.
- **`data/`** — per-job working directories (temporary; cleaned up after download or 1 hour).
- **`package-lock.json`** — regenerated by `npm install`.

## How a video gets made

1. Extension calls `POST /generate` with steps + OpenAI API key
2. Server creates a job directory under `/tmp/stepwise-video/<jobId>/`
3. For each step: writes the merged screenshot as PNG, calls OpenAI TTS, renders a segment with FFmpeg
4. Concatenates all segments into `final.mp4`
5. Calls `uploadToGHL()` → file lands in the configured GHL folder
6. Extension polls `/status/<jobId>` until done, then downloads or uses the GHL URL

## How to deploy a change to this folder

This folder is **not yet wired into the auto-deploy webhook** (as of this
commit — the webhook only deploys the publish server at `/server/`). To push
a change to the live video server today, the file must be copied onto the
VPS manually at `/opt/stepwise-video/`.

## Key lessons (see also `GHL_MEDIA_UPLOAD.md` in the claude-playbook repo)

- **GHL uploads must use `curl` via `execSync`** — Node.js native `FormData`/`fetch` fails with "Unexpected end of form"
- **`parentId` in upload form-data is the only way to place files in folders** — post-upload move endpoints all return 404
- **Folder IDs are stored as `_id` (with underscore)**, not `id`
- **Always include `Version: 2021-07-28` header** on every GHL API call
- **The merged screenshot must be used** (via `getMergedImage()` in the extension) — raw `step.screenshot` sends pre-annotation pixels and renders videos without the user's edits
