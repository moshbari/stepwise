StepWise Extension Downloads

All versions of the StepWise Chrome extension. Download, unzip, and load in Chrome.

How to Install

  1. Download the zip file below
  2. Unzip it to a folder on your computer
  3. Open Chrome and go to chrome://extensions
  4. Enable Developer mode (top right toggle)
  5. Click Load unpacked and select the unzipped folder
v3.0.0 LATEST
May 1, 2026

Annotation Editor Overhaul

  • Blur tool now shows a real Gaussian blur on the editor canvas (matching the exported result) instead of a stylized stripey overlay
  • Blur boxes are now fully editable after creation — click an existing blur with the blur tool to select it, then drag the intensity slider to change its blurriness on the fly
  • Default blur intensity dropped to 3px (the lightest setting). Dial it up only when you need more redaction
  • Step number colors no longer drift to white when the badge is selected, edited, or deleted — the color picker now controls the badge fill (bgColor) instead of the white text fill

Position Drift Fix

  • Annotation positions are now stable across save / reload cycles at any window size. Coordinates are stored in natural-image pixel space (relative to the screenshot's intrinsic dimensions) and rescaled to whatever display size the canvas is using when the file is reopened
  • In-session resize (window or sidebar) also rescales existing annotations so they stay anchored to the same point on the screenshot
  • Project file format bumped to 1.5; older files load with a graceful fallback

v2.4.6
Apr 24, 2026

Critical Fix

  • Cloud projects now stay fully editable after save and reload — circles, rectangles, arrows, callouts, step numbers, blur boxes, and pen strokes are preserved as editable objects instead of being flattened into the screenshot
  • Published HTML pages now embed the raw project data as a hidden JSON block, so Import from Published Guide also restores fully editable annotations (was previously baked pixels)
  • Live canvas shapes are now force-synced into step data on every save path, so no in-progress drawing can be lost on a fast save-to-cloud
  • Defensive recovery: if the cloud endpoint ever returns a project with empty annotations (old project, or payload truncation), the extension now silently falls back to fetching the published HTML and recovering the editable state from it
  • After updating, open any existing cloud project and click Publish once — that writes the embedded data into its HTML and makes future loads fully editable
